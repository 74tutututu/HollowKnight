# 空洞骑士复刻版项目技术说明文档

## 1. 项目综述

### 1.1 项目背景

本项目为期末课程设计——《空洞骑士》(Hollow Knight) 复刻版，由2人团队（赵晴、徐千顺）独立开发，符合"自选题目≤4人"规定。项目完全自主开发，除使用开源Cocos2d-x游戏引擎和音美素材外，所有代码均为原创，严格遵循学术诚信要求，无任何抄袭行为。

### 1.2 技术栈概述

• 引擎框架: Cocos2d-x (C++版本)  
• 编程语言: C++ (C++14标准)  
• 开发平台: Visual Studio 2022  
• 目标平台: PC (Windows) + Android (移植版)  
• 版本控制: Git分布式版本控制系统

## 2. 系统架构设计

### 2.1 核心类结构关系

项目采用面向对象设计，主要类层次结构如下：

```
Scene (游戏场景)
├── Player (主角控制器)
│   ├── PlayerState (抽象状态基类)
│   │   ├── StateIdle (待机状态)
│   │   ├── StateRun (奔跑状态) 
│   │   ├── StateJump (跳跃状态)
│   │   ├── StateSlash (攻击状态)
│   │   ├── StateCast (施法状态)
│   │   ├── ......(上劈、下劈、施法、受击、死亡等状态)
│   │   └── StateFocus (凝聚状态)
│   ├── PlayerStats (数值组件)
│   └── PlayerAnimator (动画、音效组件)
├── GameEntity (游戏实体基类)
│   ├── Enemy (地面敌人)
│   ├── Zombie (僵尸敌人)
│   ├── Buzzer (飞行敌人)
│   ├── Jar(罐子)
│   └── Spike (尖刺陷阱)
├── Boss (最终BOSS)
└── HUDLayer (UI界面层)
```

### 2.2 设计模式应用

#### 状态机模式 (State Machine Pattern)

实现方式：定义抽象基类PlayerState，各具体状态继承并实现三个核心方法：

```cpp
class PlayerState {
public:
    virtual void enter(Player* player) = 0;    // 状态进入
    virtual void update(Player* player, float dt) = 0;  // 状态更新
    virtual void exit(Player* player) = 0;     // 状态退出
};
```

#### 工厂模式 (Factory Pattern)

所有游戏对象都使用create()静态工厂方法管理生命周期，符合Cocos2d-x内存管理规范：

```cpp
Player* Player::create(const std::string& filename) {
    Player* player = new (std::nothrow) Player();
    if (player && player->init()) {
        player->autorelease();  // 托管给引擎
        return player;
    }
    CC_SAFE_DELETE(player);
    return nullptr;
}
```

#### 组件化模式 (Component Pattern)

Player类采用组件化设计，避免单一类过于庞大：
• PlayerStats：负责血量、魂量等数值管理  
• PlayerAnimator：负责动画播放和特效管理  
• PlayerState：负责行为逻辑状态管理

### 2.3 版本控制与协作流程

项目使用Git进行版本管理，采用Feature Branch工作流：
• 主分支(main)：稳定可发布版本  
• 功能分支(feature/xxx)：各成员独立开发功能  
• 提交规范：每次commit包含明确的功能描述，两名成员均有代码贡献记录  
• 代码审查：重要功能合并前进行Code Review

## 3. 核心功能与技术实现

### 3.1 Level 1 - 基础战斗系统

#### 敌方AI状态行为设计

每个敌人都继承自GameEntity基类，实现独立的AI逻辑：

##### Enemy巡逻AI：

• 状态：PATROL → ALERT → ATTACK → DEAD  
• 距离检测：使用AABB碰撞检测算法判断玩家是否在警戒范围内  
• 寻路逻辑：简单的左右巡逻，碰到边界自动转向

##### Zombie追击AI：

```cpp
bool isPlayerInChaseRange(const Vec2& playerPos) {
    float distance = abs(playerPos.x - this->getPositionX());
    return distance <= _chaseRange;
}
```

##### Buzzer飞行AI：

• 采用向量计算实现平滑跟踪  
• 垂直悬浮 + 水平追踪的复合运动模式

#### 物理碰撞检测优化

使用高效的AABB(Axis-Aligned Bounding Box)算法：

```cpp
bool intersects = (rect1.getMaxX() >= rect2.getMinX() && 
                  rect1.getMinX() <= rect2.getMaxX() && 
                  rect1.getMaxY() >= rect2.getMinY() && 
                  rect1.getMinY() <= rect2.getMaxY());
```

### 3.2 Level 2 - 解谜系统

#### 事件分发机制

梦之钉解谜系统采用基于Lambda表达式的事件分发：

```cpp
auto handleDreamHit = [&] (GameEntity* entity) {
    if (entity && entity->isValidEntity()) {
        if (_player->getDreamNailHitbox().intersectsRect(entity->getHitbox())) {
            entity->onDreamNailHit();  // 多态调用
            _player->setDreamNailActive(false);
        }
    }
};
```

#### 解谜逻辑实现

• 三个罐子分别设置不同的梦语内容  
• 通过Tag系统标识正确罐子(888号)  
• 破坏正确罐子后动态生成"复仇之魂"道具

### 3.3 Level 3 - Boss战系统

#### 3.3.1 Boss技能系统

Boss具备多阶段攻击模式：  
• 跳跃砸地：使用定时器控制攻击间隔  
• 火球弹幕：通过回调函数动态生成投射物  
• 冲击波：地面范围攻击，具备精确Hitbox判定

#### 3.3.2 定时器与回调处理

使用现代C++的std::function实现灵活的回调机制：

严格遵循Cocos2d-x内存管理规范，确保程序健壮性：

